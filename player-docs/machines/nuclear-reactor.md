# 核反应堆（Nuclear Reactor）

核反应堆是 IC2 最高级的发电设备，也是最危险的机器。它把铀燃料棒的反应热转化为电力（EU模式）或热能输出（流体模式），能量密度远超其他发电机。

- **电压等级**：Tier 5
- **最大输出**：8,192 EU/t
- **能量缓存**：1,000,000 EU
- **工作周期**：每 20 tick 完成一次反应
- **堆温上限**：10,000 HU（放入隔板可以增加上限）

---

## 搭建方法

核反应堆是一个**多方块结构**，核心有六面扩展能力，流体模式下还需要建造外壳。

### 1. 核心 + 反应仓（必备）

```
      [仓]
[仓]  [堆]  [仓]
      [仓]
```

- **中心**：核反应堆方块（`nuclear_reactor`）
- **扩展**：六个面（上下前后左右）各可接触 0 或 1 个反应仓（`reactor_chamber`）
- 每个反应仓增加 **9 格**燃料槽容量

| 反应仓数量 | 总燃料槽数 |
|-----------|-----------|
| 0 个 | 27 格 |
| 1 个 | 36 格 |
| 2 个 | 45 格 |
| 3 个 | 54 格 |
| 4 个 | 63 格 |
| 5 个 | 72 格 |
| 6 个 | **81 格** |

反应仓紧贴核反应堆放置即可，不需要额外连接。右键点击反应仓会自动打开反应堆的 GUI。

### 2. 流体模式外壳（可选，流体模式专用）

如果只想要发电，只需要核心 + 反应仓就够了，不需要外壳。这叫**EU模式**。

如果想使用**流体模式**（产热翻倍 + 生产热冷却液），需要建造一个 **5×5×5** 的外壳把反应堆整个包起来。

```
外壳示意图（水平剖面，反应堆在中心）：
Layer 0 & 4（最外层）：全部外壳方块
Layer 1-3（中间层）：
  #####         # = 外壳方块
  #   #         . = 内部空间（3×3×3 空心）
  # . #         
  #####

实际尺寸：5 格宽 × 5 格高 × 5 格深
反应堆在正中心 (2, 2, 2)
内部 3×3×3 是空心区域
```

外壳方块可以使用以下四种：

| 方块 | 用途 |
|------|------|
| `reactor_vessel`（压力容器） | 外壳主体，主要填充材料 |
| `reactor_access_hatch`（访问接口） | 用管道自动装卸燃料棒 |
| `reactor_fluid_port`（流体接口） | 输入冷却液 + 输出热冷却液，**流体模式至少需要 2 个** |
| `reactor_redstone_port`（红石接口） | 红石信号控制启停 |

> 流体模式要求外壳上至少安装 **2 个流体接口**（一个进冷却液，一个出热冷却液）。

### 3. 布局锁定功能

反应堆支持**布局锁定**，锁定后只有同种物品才能放入对应槽位，空槽位不能放入任何物品。

锁定主要用于物流自动化（如 AE2 自动更换燃料棒）：解锁 → 全部抽出 → 全部放入新棒 → 锁定，**一次完成换棒**，不需要每 tick 只换一种。在强制冷却核电方案中尤其重要——换棒速度直接关系反应堆安全。

在 GUI 中点击锁定按钮即可启用/解除锁定。

## 燃料棒机制

### 基础原理

反应堆使用**铀燃料棒**和**MOX 燃料棒**发电。燃料棒在反应堆 GUI 的网格中摆放，每格只能放 1 个。

- 燃料棒**不需要消耗燃料**，而是有**耐久度**（脉冲数）
- 每个工作周期（20 tick）消耗 1 脉冲
- 燃料棒会**产热**（HU）同时**发电**（EU）
- 相邻的燃料棒会互相增强——这叫"邻接加成"

### 相邻增强效应

燃料棒越密集，每一根的效率越高。这就是为什么反应仓越多越好——可以塞更多燃料棒形成密集排布。

- 一根燃料棒每相邻另一根燃料棒，脉冲数增加
- 四联燃料棒自身就相当于 4 根捆在一起，邻接还会进一步强化
- **中子反射器**可以贴在燃料棒旁边，模拟"有一根燃料棒相邻"的效果，不消耗耐久

### 燃料棒类型

| 燃料棒 | 基准发电 | 发热量 | 耐久（脉冲数） |
|--------|---------|--------|---------------|
| 单联铀燃料棒 | 100 EU/t | 4 HU/脉冲 | 20,000 |
| 双联铀燃料棒 | 400 EU/t | 4 HU/脉冲 | 20,000 |
| 四联铀燃料棒 | 1,200 EU/t | 4 HU/脉冲 | 20,000 |
| 单联 MOX 燃料棒 | 100~500 EU/t（随堆温升高） | 4 HU/脉冲 | 10,000 |
| 双联 MOX 燃料棒 | 400~2,000 EU/t | 4 HU/脉冲 | 10,000 |
| 四联 MOX 燃料棒 | 1,200~6,000 EU/t | 4 HU/脉冲 | 10,000 |

MOX 燃料棒的发电量随堆温升高而线性增加。堆温 10,000 HU 时达到最大效率（约 5 倍）。这意味着 MOX 适合在高温下运行，需要配合强大的散热系统。

### 散热片

燃料棒运行时会产热，热量必须被散走，否则堆温会不断升高直至爆炸。散热片是管理堆温的关键。

| 散热片 | 热容量 | 自身蒸发/周期 | 吸堆温/周期 |
|--------|--------|--------------|------------|
| 散热片 | 1,000 HU | 6 HU | 0 HU |
| 反应堆散热片 | 1,000 HU | 5 HU | 5 HU |
| 元件散热片 | 0 HU | 0 HU | 0 HU |
| 高级散热片 | 1,000 HU | 12 HU | 0 HU |
| 超频散热片 | 1,000 HU | 20 HU | 36 HU |

- **自身蒸发**：散热片自己蒸发到空气中的热量。这部分热量在EU模式下直接降低堆温，在流体模式下被用来加热冷却液
- **吸堆温**：散热片从反应堆堆温中抽走的热量
- 散热片有热容量上限，满了会损坏

### 热交换器

热交换器在燃料棒、散热片和堆温之间重新分配热量。

| 热交换器 | 热容量 | 邻接交换 | 堆交换 |
|---------|--------|---------|--------|
| 热交换器 | 2,500 HU | 12 HU | 4 HU |
| 反应堆热交换器 | 5,000 HU | 0 HU | 72 HU |
| 元件热交换器 | 5,000 HU | 36 HU | 0 HU |
| 高级热交换器 | 10,000 HU | 24 HU | 8 HU |

### 冷却单元

冷却单元是纯吸热组件，自身没有耐久，能吸收大量热量。

| 冷却单元 | 热容量 |
|---------|--------|
| 反应堆冷却单元 | 10,000 HU |
| 三联反应堆冷却单元 | 30,000 HU |
| 六联反应堆冷却单元 | 60,000 HU |

### 隔板

隔板不参与热交换，但能**提高堆温上限**：

| 隔板 | 堆温上限加成 | 爆炸修正 |
|------|------------|---------|
| 反应堆隔板 | +500 HU | ×0.9 |
| 高热容反应堆隔板 | +1,700 HU | ×0.99 |

### 中子反射器

贴在燃料棒旁边可以增加脉冲数，自身不产热不发电：

| 反射器 | 脉冲容量 | 耐久 |
|--------|---------|------|
| 中子反射板 | 30,000 脉冲 | 有限 |
| 加厚中子反射板 | 120,000 脉冲 | 有限 |
| 铱中子反射板 | 无限 | 无限 |

## 两种工作模式

反应堆有两种截然不同的工作模式，由是否建造外壳自动决定。

### EU模式（无外壳）

最直接的用法。不需要建造 5×5×5 外壳：

- 燃料棒产热 → 散热片降温 → 堆温管理 → 直接发电
- 产热量正常
- 输出的 EU 通过能量端口传出，最高 8,192 EU/t
- 堆温越高危险越大，需要用散热片控制温度

### 流体模式（有外壳 + 冷却液）

建造 5×5×5 外壳并接入冷却液后自动进入流体模式：

- **产热翻倍**
- **不直接发电**——热量被用来加热冷却液
- 散热片的蒸发热量会转化为热能：**20,000 HU → 1 桶热冷却液**
- 热冷却液可以送到斯特林发电机等热机发电，或用于其他需要热能的机器
- 冷却液储罐容量：输入输出各 **16,000 mB**

> 流体模式的核心思路：用液体把热量搬运出去，在更安全的地方转化为电力。代价是建造外壳和铺设管道的成本，换来更高的能量产出。

#### 热量守恒规则

```
燃料棒产热 × 2
    ↓
散热片蒸发出一部分热量（散失热量）
    ↓
散失热量 → 加热冷却液 → 热冷却液输出
    ↓
未用完的热量 → 回灌到散热片 / 最终回到堆温
```

## 堆温与环境风险

反应堆的温度（堆温）范围是 0~10,000 HU。温度越高，风险越大。

### 温度阈值

| 堆温 | 后果 |
|------|------|
| > 4,000 HU | 附近方块**可能着火** |
| > 5,000 HU | 附近水源**可能蒸发** |
| > 7,000 HU | 范围内生物受到**辐射伤害**（防化服可免疫） |
| > 8,500 HU | 附近方块**可能变成岩浆** |
| ≥ 10,000 HU | **核爆炸** |

> 防化服（头盔 + 胸甲 + 护腿）可以完全免疫 7,000 HU 以上堆温的辐射伤害。

### 核爆炸

当堆温达到 10,000 HU 时：
1. 爆炸威力根据堆温、隔板数量、燃料棒种类计算（上限 100）
2. 摧毁反应堆和所有相邻反应仓
3. 范围内方块被夷平
4. 全服公告：所有者、位置、破坏方块数量
5. 同一玩家 30 分钟内不会重复公告

爆炸可以通过配置文件关闭（改为无害融毁）。

### 温度自动效果

反应堆每个工作周期会检查堆温并施加环境影响：

- 着火：5×5×5 范围内随机 2% 概率点燃
- 蒸发水：5×5×5 范围内随机 2% 概率蒸发水源
- 岩浆化：5×5×5 范围内随机 1% 概率将固体方块变岩浆（保护：基岩、反应堆、机器、线缆不转换）
- 领地保护：受 FTB Chunks 保护的领地不受以上效果影响

## 配套方块一览

核反应堆共有 **6 种** 相关方块：

### 核反应堆（`nuclear_reactor`）

核心方块。Tier 5 等级，GUI 中有燃料槽网格（27 ~ 81 格）、温度显示、锁定按钮。

### 反应仓（`reactor_chamber`）

贴在反应堆六个面上各可贴 1 个，每增加一个反应仓增加 9 格燃料槽。右键反应仓 = 打开反应堆 GUI。自身无独立功能。

### 压力容器（`reactor_vessel`）

5×5×5 外壳的主要填充方块，纯结构方块，无 Tile Entity。放在外壳层的指定位置即可。

### 访问接口（`reactor_access_hatch`）

流体模式外壳的组成部分之一，同时也是**物品自动装卸口**：

- 可以用管道向其输入/输出物品，自动委托给中心反应堆的 Inventory
- 右键可以打开反应堆 GUI
- 主要用于自动化装卸燃料棒和枯竭燃料棒

### 流体接口（`reactor_fluid_port`）

流体模式外壳的组成部分之一，**流体模式至少需要 2 个**：

- 输入冷却液 → 反应堆冷却液储罐
- 输出热冷却液 → 管道系统
- 自带 1 个升级槽：可安装流体抽入/弹出升级，实现自动化流体传输
- 右键可打开流体接口配置界面

### 红石接口（`reactor_redstone_port`）

流体模式外壳的组成部分之一，用于红石控制：

- 接收红石信号控制反应堆启停
- 支持信号**反转**（信号高 = 停止，信号低 = 运行）
- 同一个反应堆可以安装多个红石接口，**所有接口都允许运行时反应堆才会运行**
- 如果反应堆上没有安装任何红石接口，则使用反应堆自身的红石控制设置

## 红石控制

反应堆支持精细的红石控制：

1. **直接给反应堆方块施加红石信号**：信号高时停止/运行（可在 GUI 中切换反转）
2. **安装红石接口**：当外壳上装有红石接口时，红石接口的信号取代反应堆自身的红石控制
3. **多接口逻辑**：多个红石接口的允许运行状态**取与**——全部允许时反应堆才运行

这个功能可以用来做安全连锁：例如用一个红石接口接温度监控器，温度超标时自动停堆。

## 自动化建议

### 基础方案：EU模式发电

1. 核心 + 6 个反应仓（81 格）
2. 排布燃料棒 + 散热片 + 热交换器
3. 用 Tier 5 导线（或变压器降压）引出电力到储电箱
4. 定期手动更换枯竭燃料棒

### 进阶方案：流体模式 + 斯特林发电机

1. 建造 5×5×5 外壳 + 至少 2 个流体接口
2. 外壳内布置燃料棒和散热片
3. 冷却液管道 → 流体接口（输入）
4. 热冷却液 → 斯特林发电机 → 发电
5. 斯特林发电机出来的冷却液冷却后循环回去

### 全自动方案

- 访问接口 + 管道：自动输入新燃料棒、抽出枯竭燃料棒
- 流体接口 + 流体抽入/弹出升级：自动循环冷却液
- 红石接口 + 温度监控：过热自动停堆保护
- 布局锁定：防止管道在装卸时搞乱排布

## 安全须知（必读）

1. **别让堆温到 10,000**——这真的会炸，不是开玩笑
2. **流体模式并不更安全**——只是把热量转移到冷却液中，堆温管理仍然至关重要
3. **建在远离基地的地方**——或者用防爆墙隔离
4. **首次启动时蹲在旁边观察**——看堆温是否能稳定在安全范围
5. **布局锁定是你的朋友**——用管道自动装卸时，锁定可以防止空槽位被塞入错误的物品
6. **防化服**在 7,000 HU 以上时保命，进高温反应堆区域务必穿着
