# 区块加载器 (Chunk Loader)

## 基础信息

- **电压等级**：Tier 1 (LV)
- **能量缓存**：10,000 EU
- **耗能**：1 EU/t 每区块
- **最大输入**：32 EU/t
- **最大加载范围**：5×5 区块网格（共 25 区块）
- **升级槽**：无（不支持任何升级）

## 怎么用

区块加载器可以强制加载周围的区块，让机器和作物在你离开后也能继续工作。它会在机器所在区块为中心、半径 2 区块的 5×5 范围内加载区块。

### 选择要加载的区块

放置并右键打开 GUI，你会看到一个 5×5 的区块网格。每个格子对应一个区块，点击格子可以切换该区块的加载状态：

- **绿色**：已加载（该区块会被强制保持活跃）
- **灰色**：未加载

#### 中心区块

网格最中心的区块（第 12 号，即第 3 行第 3 列）**始终处于加载状态**，无法关闭——因为机器本身就在这个区块里，不加载它机器自己就没法运行了。

#### 选择策略

一次性加载全部 25 个区块最省事，但开销也最大（25 EU/t）。你完全可以只加载自己需要的区块来省电，比如：

- 水平产线沿着一条直线延伸的，只需加载一行（5 个区块）
- 只加载你经常活动的核心区域（9 个区块中心区域）
- 搭配实际需要，只勾选有机器运作的那几个区块

### 电池槽

机器左上角有一个电池槽，支持放入放电电池来提供能量。把有电的电池放进去，机器会自动抽取电量。

## 能量 supply

区块加载器**只消耗能量**，不产生也不输出能量。

| 途径 | 说明 |
|------|------|
| 导线输入 | 从相邻导线或储能设备接收 EU，最大 32 EU/t |
| 电池供电 | 放入电池槽的电池会自动放电给机器 |

机器内部缓存 10,000 EU。满负载（25 区块）下，缓存约能支撑 6 分 40 秒（10,000 ÷ 25 = 400 秒）。

### 能量耗尽

如果内部缓存不足以支付当前 tick 的加载费用，区块加载器会**立即释放所有已加载的区块**，机器恢复为未激活状态。一旦重新获得足够能量，它会自动恢复加载。所以如果发现区块加载器总是一闪一闪的，说明它电力不足——给它多供点电，或者减少加载的区块数。

## 区块加载工作机制

区块加载器使用 Minecraft 原生的强制加载机制（ChunkTicketType.FORCED），这意味着：

- 被加载的区块会保持**全部活跃**：实体（Entity）会正常 tick，作物生长，机器持续运作
- 服务器重启后，你需要重新放置并启动区块加载器（配置不保留在世界上）
- 在没有玩家在线时，区块依然保持加载

### 挖掉机器

用任何工具破坏区块加载器时，它会在掉落前自动释放所有加载的区块。不用担心留下"幽灵区块"。

## 没有的能力

- **不支持升级**：不能装超频、储能、变压器等任何升级，它的性能是固定的
- **不支持输出能量**：这是一个纯耗能设备，不向外输出 EU
- **不支持流体**：不能处理任何液体

## GUI 界面

| 元素 | 说明 |
|------|------|
| 左上角格子 | 电池放电槽 |
| 中间 5×5 网格 | 区块选择面板，点击切换加载状态 |
| 底栏 | 当前加载区块数量及耗能信息 |

GUI 底栏显示三项信息：
- **加载范围**：当前选中的区块数量
- **耗能**：形如"25.0 EU/t (25 区块)"
- **各区块状态**：绿色/灰色标识加载状态

## 建造建议

1. 确保供电充足。一台 LV 变压器（或 BatBox/MFE）配合导线可以稳定供应 32 EU/t，足以支持满负载运行
2. 不需要加载全部 25 个区块时，在 GUI 中关闭不必要的区块以节省电力
3. 区块加载器本身不提供备用电源，建议连接到稳定的电网或储能设备
4. 不要只依赖电池槽供电——那只适合短时间使用或应急，长时间运行还是拉导线更靠谱

## 常见用途

- **保持矿机持续运作**：配合矿机，离线后自动挖矿
- **保持作物/树场生长**：加载区块使作物继续生长
- **保持生产线运作**：让流水线在你下线后持续产出
- **跨维度加载**：配合传送门，在主世界加载下界或末地的区块

## 交互

- **右键**：打开 GUI，选择要加载的区块
- **扳手右键**：旋转机器（改变朝向，但区块加载器没有输入/输出面之分，旋转纯装饰）
- **扳手左键**：瞬间拆下机器，完好掉落（耗耐久）
