# 传送机

<BlockImage id="ic2_120:teleporter" p:facing="north" scale="4" />

传送机会把附近的一名玩家、生物或其他实体传送到同维度内已配对的另一台传送机。它不会为传送本身储存电量：起点传送机会直接从相邻的 MFE、MFSU、MFE 充电板或 MFSU 充电板中扣除本次传送所需 EU。

## 配对

使用遥控器（Frequency Transmitter）配对两台传送机。

1. 潜行右击，或用尚未记录目标的遥控器右击第一台传送机，记录它的位置。
2. 在同一维度右击第二台传送机。
3. 两台传送机会互相写入对方坐标。

目标位置在使用时仍必须是有效传送机，必须与起点在同一维度，并且目标区块必须已加载。传送机不能绑定到自身。

## 供能

- **机器等级**：4
- **必需相邻储能方块**：MFE、MFSU、MFE 充电板或 MFSU 充电板
- **传送用内部缓存**：无

起点会先统计所有相邻有效储能方块中的 EU。电量足够时开始充能；充能完成后，再从这些相邻方块中扣除实际费用。目标传送机不支付本次费用，但它必须有效且不在冷却中。

## 触发

传送机只有在收到红石信号时才会开始传送。收到信号后，它会扫描激活范围，并选择最近的有效实体。实体必须存活，不能正在骑乘，不能带有乘客，并且充能期间不能离开激活范围。

在传送机 GUI 中可以切换激活范围：

- **1x1x1**：传送机上方的紧凑触发范围
- **3x3x3**：适合平台、生物或自动化布置的较大触发范围

传送成功后，起点和目标传送机都会进入 20 tick 冷却。

## 消耗与充能时间

费用由实体重量和两台传送机方块之间的直线距离决定：

`EU = floor(5 * 重量 * (距离 + 10)^0.7)`

重量规则：

- **玩家**：基础 1000，每件已装备护甲 +100，并按物品栏堆叠占用增加重量，总重量最高 5100
- **动物**：100
- **怪物或其他实体**：500

充能时间为 `40 + 距离 / 20` tick，并限制在 40 到 120 tick 之间。两个升级槽中的超频升级可以缩短传送前充能时间，但最低仍为 40 tick。

## 槽位与自动化

GUI 有两个升级槽，可放入升级物品；以当前传送机行为来说，超频升级会影响传送前的充能时间。自动化物品输入只会路由到这两个升级槽，也允许从这些槽中取出物品。

## 实用布置

把 MFE、MFSU 或对应充电板直接贴在起点传送机旁并保持有电，用遥控器完成配对，然后用红石信号驱动起点。若需要双向往返，两端都要供电并放置相邻储能方块，因为从哪一端发起传送，哪一端就负责支付 EU。
