# 作物生长环境需求

本文档汇总当前实现中各作物（种子）的生长环境要求，便于测试与平衡。

关联源码：
- `src/main/kotlin/ic2_120/content/block/CropBlock.kt`（`canGrowAt` 与专属行为）
- `src/main/kotlin/ic2_120/content/crop/CropSystem.kt`

## 说明

- “基础种子”列表示可直接在作物架上右键种植的原版物品。
- “仅杂交”表示无基础种子，通常通过杂交获得。
- 金属作物在末阶段前一龄额外需要根系方块条件（见文末）。

## 停止生长的常见原因（先看这个）

如果作物长期不长，优先按这个顺序排查：

1. 是否达到该作物的硬条件（光照、地块、专属条件）。
2. 是否已经到该作物的成熟年龄上限。
3. 是否是金属作物且缺少下方根系方块。
4. 营养/水分/空气质量是否过低导致生长效率极差（看起来像“停长”）。

## 作物采摘与掉落规则（右键采摘）

关联源码：
- `src/main/kotlin/ic2_120/content/block/CropBlock.kt`（`canBeHarvested`、`performHarvest`、`createGainStacks`）
- `src/main/kotlin/ic2_120/content/crop/CropSystem.kt`（`canBeHarvested`、`dropGainChance`、`dropSeedChance`、`ageAfterHarvest`）

### 1) 何时可以右键采摘

- 默认：`age >= 该作物成熟龄上限` 才可采摘。
- 例外：
  - `reed / sticky_reed`：`age > 0` 即可采摘。
  - `venomilia`：`age >= 3` 即可采摘。
  - `coffee / potato`：`age >= maxAge - 1` 即可采摘。

### 2) 会掉什么（总体）

- 每次采摘由“产物掉落轮数”决定产物数量，轮数受作物 Tier 与种子 `Ga(增益)` 影响。
- 每轮会生成该作物的产物堆；当 `Ga` 较高时，单堆有机会额外 `+1`。
- 另外独立判定一次种子袋掉落（不是每轮都掉种子袋）。

### 3) 种子袋掉率

- 基础公式（当前实现）：
  - 初始 `0.5`
  - 若当前 `age == 1`，再乘 `0.5`
  - 再按作物 `Tier` 每级乘 `0.8`
- 即：`chance = 0.5 * (age==1 ? 0.5 : 1.0) * (0.8 ^ tier)`
- 命中后掉落当前作物种子袋，继承当前三维与扫描等级。

### 4) 采后回龄（可连收）

- 回龄到固定年龄的作物可继续长并再次采摘（可连收）。
- 回成作物架（`null`）的作物需要重新种植。

## 肥料、水分、群系、机器污染（定性分析）

以下几项不一定直接触发“禁止生长”，但会显著改变生长速度，甚至造成看似停长。

### 1) 肥料（营养，`nutrients`）

- 手动施肥：每次约 `+100 nutrients`（在 `<100` 时可施加）。
- 监管机施肥：每次约 `+90 nutrients`。
- 作物每个主生长周期会自然消耗一点营养。

影响：

- 营养会提升“地形营养值”和环境加成，直接提高生长点积累速度。
- `cocoa` 有硬门槛：`nutrients >= 3` 才能继续长。
- 金属作物对营养更敏感，低营养时会明显变慢。

### 2) 水分（湿度，`humidity`）

湿度主要来自：

- 是否下雨；
- 群系温度是否在适宜区间；
- 下方耕地是否有湿润值；
- 作物内部储水（人工补水/监管机补水）。

影响：

- 湿度越高，生长质量通常越高。
- `reed/sticky_reed` 这类高需水作物对水分加成更敏感。
- 长期缺水不会立刻“死亡”，但会把生长拖得非常慢。

### 3) 群系（主要影响温度与间接湿度/营养）

- 温度会参与营养与湿度计算。
- 过冷/过热群系可能让部分地块天然生长质量偏低。

建议：

- 量产田优先选中性环境；
- 极端群系里要更多依赖监管机补水补肥。

### 4) 机器污染（空气质量，`air quality`）

空气质量受周围“开阔度”影响：

- 作物周围过多不透明方块会降低空气质量；
- 周围作物过于拥挤也会拉低开阔度；
- 头顶见天通常会提供明显正向加成。

“机器污染”的玩家理解可以这样记：

- 把大量机器、实体方块紧贴农田堆放，会让空气质量变差，进而拖慢作物。
- 农田区与工业区最好留一点间距，或者至少保证田区上方开阔。

## 理论平均长成时间（标准环境）

可以给出“每种作物平均长成时间”，但必须先固定环境参数。  
下表使用的是当前实现的一组标准假设值（用于横向比较）：

- 三维固定为初始值：`G=1, Ga=1, R=1`
- 田块为湿润耕地、上方开阔、可见天
- 中性群系温度、无额外手动补水/施肥、无监管机加速
- 只统计从 `age=0` 到该作物最大成熟龄的理论平均时间
- 已把生长公式中的随机项按数学期望处理

注意：

- 这是“理论均值”，实战会因光照、群系、水肥、Weed-EX、机器布局、特殊行为而波动。
- `nether_wart/terra_wart` 的底材加速（灵魂沙/雪块）未计入这张基准表，实际可更快。
- 某些作物若硬条件不满足，会直接停长（不是“慢”，而是“不再推进年龄”）。

## 金属作物根系方块校验

在 `ferru/cyprium/stagnium/plumbiscus/aurelia/shining` 的“末阶段前一龄”时，会向下扫描 `1..5` 格，要求至少存在一个匹配方块：

- `ferru`：铁矿/铁块（`minecraft` 与 `c:ores/iron`、`c:storage_blocks/iron`）
- `cyprium`：铜矿/铜块（`minecraft` 与 `c:ores/copper`、`c:storage_blocks/copper`）
- `stagnium`：锡矿/锡块（`c:ores/tin`、`c:storage_blocks/tin`）
- `plumbiscus`：铅矿/铅块（`c:ores/lead`、`c:storage_blocks/lead`）
- `aurelia`：金矿/金块（`minecraft` 与 `c:ores/gold`、`c:storage_blocks/gold`）
- `shining`：银矿/银块（`c:ores/silver`、`c:storage_blocks/silver`）

## 玩家速查：作物类型 -> 推荐环境与管理策略

这部分是“怎么玩更稳更快”的攻略版结论。

| 作物类型 | 推荐地形/区位 | 光照建议 | 管理重点 | 常见停长点 |
|---|---|---|---|---|
| 主粮类（wheat/carrot/beetroots/pumpkin/melon/potato） | 普通平原耕地，尽量露天 | 保持 `>=9` | 维持基础水肥即可 | 光照不足；马铃薯到上限龄后不再继续 |
| 花类（dandelion/poppy/blackthorn/tulip/cyazint） | 开阔、无遮挡田区 | 保持 `>=12` | 重点保证高光，不必极限施肥 | 常被遮挡导致亮度不够 |
| 芦苇系（reed/sticky_reed） | 靠近集中供水区，方便监管机覆盖 | 保持 `>=9` | 优先补水，其次补肥 | 缺水时生长明显变慢 |
| 可可/咖啡/啤酒花（cocoa/coffee/hops） | 中性群系，监管机覆盖区 | 保持 `>=9` | 持续供肥；可可注意营养门槛 | 可可在低营养（`<3`）会直接卡住 |
| 蘑菇/疣类（red_mushroom/brown_mushroom/nether_wart/terra_wart） | 单独分区，避免和高光作物混田 | 不依赖高光（疣类看底材更关键） | 地狱疣配灵魂沙、大地疣配雪块可提速 | 底材没配对时效率低，且存在互转行为 |
| 树苗类（oak/spruce/birch/jungle/acacia/dark_oak） | 开阔露天，少遮挡 | 保持 `>=9` | 常规水肥维护 | 顶部或侧面遮挡导致慢长 |
| 金属类（ferru/cyprium/stagnium/plumbiscus/aurelia/shining） | 专门矿物温室（下方预埋对应矿/块） | 常规即可 | 保持营养中高位，提前布好根系方块 | 末阶段前一龄缺根系方块会停长 |
| 红小麦（red_wheat） | 独立区，方便控光 | 严格 `5..10` | 用半遮光或固定光源控制亮度 | 光太高或太低都会卡住 |
| 特殊攻击类（venomilia/eating_plant） | 独立隔离区，远离常用通道 | 按各自硬条件 | 以观察/育种为主，不建议混入量产田 | 条件苛刻，且会干扰实体/玩家 |

### 田区规划建议（实用）

1. 育种田与量产田分开，避免杂交区拖慢主产线。
