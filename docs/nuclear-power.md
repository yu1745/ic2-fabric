# 核电（核反应堆）容量与热量逻辑

## 多方块结构

核反应堆采用多方块结构：

- **中心**：核反应堆方块（`nuclear_reactor`）
- **扩展**：六面（上下前后左右）可各接触 0 或 1 个核反应仓（`reactor_chamber`）

所有 NBT 数据均存储在反应堆的 BlockEntity 上，反应仓本身无独立存储。

## 槽位布局

- 布局：9 行 × (3–9) 列，`index = col * 9 + row`
- 相邻：上下左右四方向 `(x±1, y)`、`(x, y±1)`

## 槽位容量公式

```
当前有效容量 = 27 + 相邻反应仓数 × 9
```

### 常量说明

| 常量 | 值 | 说明 |
|------|-----|------|
| `BASE_SLOTS` | 27 | 基础槽位，无反应仓时的容量 |
| `SLOTS_PER_CHAMBER` | 9 | 每个相邻反应仓增加的槽位 |
| `MAX_SLOTS` | 81 | 最大槽位数（6 个反应仓时） |

## 计算周期与工作流程

- **Tick 率**：20 tick（每秒一次计算）
- **Tick 偏移**：BE 构造时随机 `0..19`，当 `(world.time + tickOffset) % 20 == 0` 时执行
- **目的**：分散全服核电计算，避免同一 tick 卡顿

### processChambers 双阶段

每周期执行两次遍历：

1. **Pass 0 (heatRun=false)**：发电与中子脉冲
   - 燃料棒：`acceptUraniumPulse` 向邻接传递脉冲，`addOutput(1.0F)` 发电
   - 中子反射器：`acceptUraniumPulse` 返回 true，增加邻接燃料棒脉冲数
   - 燃料棒耐久消耗，耗尽后替换为枯竭燃料棒

2. **Pass 1 (heatRun=true)**：热量分配
   - 燃料棒：`heat = triangularNumber(pulses) * 4`，优先传给邻接可储热组件，剩余 `addHeat`
   - 散热片：吸堆温 → 自身蒸发 / 向邻接蒸发
   - 热交换器：与堆/邻接组件交换热量

### 能量输出

- `output` 为 float 累加值（每脉冲 1.0）
- 每周期结束：`output * 5.0` 转为 EU（`getOfferedEnergy()` 中计算）
- 输出上限：8192 EU/t（tier 5）

#### 铀燃料棒发电量

| 类型 | 基础脉冲数 | 循环次数 | 总脉冲数 | EU/t |
|------|----------|---------|---------|------|
| 单联铀燃料棒 | 1 | 1 | 1 | 5 |
| 双联铀燃料棒 | 2 | 2 | 4 | 20 |
| 四联铀燃料棒 | 3 | 4 | 12 | 60 |

计算公式：
- `basePulses = 1 + numberOfCells / 2`
- 总脉冲数 = `basePulses * numberOfCells`
- EU/t = `总脉冲数 * 5`

## 热量与堆温

反应堆可贮存热量，最多 **10,000 点**。贮存的热量即为**堆温**。

- 堆温范围：0 ~ 10,000
- 燃料棒发热优先传给邻接可储热组件，剩余进入堆温
- 散热片蒸发热量通过 `addEmitHeat(-evaporated)` 降低堆温
- 堆温 ≥ 10,000 时爆炸

## 堆温对环境的影响

| 堆温阈值 | 影响范围 | 效果 |
|---------|---------|------|
| > 4,000 | 5×5×5 | 方块有几率着火 |
| > 5,000 | 5×5×5 | 水有几率蒸发 |
| > 7,000 | 7×7×7 | 生物有几率受到伤害（防化服可抵挡） |
| > 8,500 | 5×5×5 | 方块有几率变成岩浆 |
| ≥ 10,000 | — | **核反应堆爆炸** |

### 防化服

穿戴完整防化服（头盔、胸甲、护腿）的生物可免疫堆温 > 7,000 时的辐射伤害。防化服物品 ID 包含 `hazmat`。注意：防护服尚未实现。

## 反应堆组件 API

- **IReactor**：反应堆接口，供组件访问槽位、热量、输出等
- **IBaseReactorComponent**：可放入反应堆的物品需实现，`canBePlacedIn`
- **IReactorComponent**：`processChamber`、`acceptUraniumPulse`、`canStoreHeat`、`alterHeat` 等
- **AbstractReactorComponent**：默认空实现
- **AbstractDamageableReactorComponent**：NBT `"use"` 存储耐久/热量

## 燃料棒

- **铀燃料棒**（单/双/四联）：发电、发热，耗尽后变为枯竭燃料棒
- **枯竭燃料棒**：占位，不可发电
- 发热量：`triangularNumber(pulses) * 4`，优先传给邻接可储热组件

## MOX 燃料棒

MOX 燃料棒（Mixed Oxide Fuel）是一种特殊的燃料棒，其发电和发热特性与堆温密切相关。

### 类型

- **单联 MOX 燃料棒**（`mox_fuel_rod`）
- **双联 MOX 燃料棒**（`dual_mox_fuel_rod`）
- **四联 MOX 燃料棒**（`quad_mox_fuel_rod`）

耗尽后变为对应的枯竭 MOX 燃料棒。

### 发电公式

MOX 燃料棒的发电量与堆温成正比：

```
堆温效率 = reactor.heat / reactor.maxHeat
每脉冲发电 = 4.0 × 堆温效率 + 1.0
```

- **最低发电**（堆温 = 0）：每脉冲 1.0 output
- **最高发电**（堆温 = 10,000）：每脉冲 5.0 output
- **中等发电**（堆温 = 5,000）：每脉冲 3.0 output

#### MOX 燃料棒发电量（单联）

| 堆温 | 堆温效率 | 每脉冲发电 | EU/t |
|------|---------|-----------|------|
| 0 | 0% | 1.0 | 5 |
| 2,500 | 25% | 2.0 | 10 |
| 5,000 | 50% | 3.0 | 15 |
| 7,500 | 75% | 4.0 | 20 |
| 10,000 | 100% | 5.0 | 25 |

计算公式：
- EU/t = `每脉冲发电 * 5`
- 总脉冲数计算与铀燃料棒相同（受邻接燃料棒影响）

### 发热公式

MOX 燃料棒在**液冷反应堆**（`isFluidCooled()` 返回 true）中有特殊的发热特性：

- 基础发热：`triangularNumber(pulses) * 4`（与铀燃料棒相同）
- **热量翻倍条件**：当 `堆温效率 > 0.5`（即堆温 > 5,000）时，发热量 × 2

**注意**：MOX 燃料棒的液冷特性与**热模式**是不同的概念：
- **MOX 液冷特性**：MOX 燃料棒在液冷环境下的固有特性，高温时发热量翻倍，但仍正常发电
- **热模式**：整个反应堆的工作模式，发热量翻倍但**不发电**，通过冷却液散热

### 特性对比

| 特性 | 铀燃料棒 | MOX 燃料棒 |
|------|---------|-----------|
| 每脉冲发电 | 固定 1.0 | 1.0 ~ 5.0（取决于堆温） |
| 单联 EU/t | 5 | 5 ~ 25（堆温 0 ~ 10,000） |
| 发热量 | `triangularNumber(pulses) * 4` | 同铀燃料棒，液冷且堆温>5000时翻倍 |
| 最佳工作温度 | 任意 | 高温（堆温越高发电越多） |
| 适用场景 | 稳定发电 | 高温高发电效率 |

### 使用建议

- MOX 燃料棒适合在高温反应堆中使用，以获得最大发电效率
- 需要配合足够的散热系统，避免堆温达到爆炸阈值
- 在液冷反应堆中，堆温超过 5,000 时发热量会翻倍，需要更强的散热能力

## 热模式（液冷反应堆）

热模式是核反应堆的另一种工作模式，与电模式相比具有完全不同的特性。

### 模式判定条件

热模式通过检测外层结构来判定：

- **结构要求**：外层必须是 5×5×5 的立方体外壳
- **外壳组成**：由反应堆压力容器（`reactor_vessel`）、红石接口（`reactor_redstone_port`）、流体接口（`reactor_fluid_port`）组成
- **中心**：核反应堆方块
- **必要条件**：至少需要一个流体接口
- **检测频率**：每 5 秒（100 ticks）重新检测一次

### 热模式与电模式的区别

| 特性 | 电模式 | 热模式 |
|------|--------|--------|
| 发热量 | 正常 | **× 2** |
| 发电量 | 正常输出 EU | **不发电** |
| 散热作用 | 降低堆温 | **加热冷却液** |
| 流体需求 | 无 | **冷却液 + 热冷却液** |
| 流体接口 | 无需 | **至少 2 个**（1 输入 + 1 输出） |
| 堆温影响 | 存在，超温爆炸 | **仍然存在，超温仍爆炸** |

### 冷却液系统

热模式需要流体系统来散热：

- **冷却液输入**：通过流体接口或手动注入冷却液
- **热冷却液输出**：散热片产生的热量用于加热冷却液
- **转换比例**：每 **20,000 HU** 散失热量可加热 **1,000 mB** 冷却液变成热冷却液
- **存储容量**：输入和输出罐各 10 buckets（10,000 mB）

### 散失热量的定义

**重要**：只有散热片的 `selfVent`（自身蒸发）才是真正的散失热量，可用于冷却液转换：

- **`selfVent`**：散热片自身蒸发热量 ✓ 用于冷却液加热
- **`reactorVent`**：从反应堆吸收热量 ✗ 不是散失热量
- **直接接触传热**：组件间的热传递 ✗ 不是散失热量

### 散热片耐久修复

散热片的耐久修复在不同模式下有不同条件：

| 模式 | 耐久修复条件 |
|------|-------------|
| 电模式 | 总是可自然修复 |
| 热模式 | **只有存在冷却液时才能修复** |

### 热模式工作流程

1. **模式检测**：每 100 ticks 检测一次外层结构
2. **燃料棒产热**：计算基础热量（与电模式相同）
3. **发热量翻倍**：在 `processChambers()` 完成后，统一将总产热量 × 2
4. **散热片工作**：
   - `reactorVent`：从堆吸收热量存储到散热片自身
   - `selfVent`：蒸发自身热量，记录为散失热量（用于冷却液转换）
   - 热模式下：只有冷却液充足时，`selfVent` 才能修复散热片耐久
5. **冷却液加热**：散失的热量将冷却液转换为热冷却液
6. **堆温更新**：剩余热量仍添加到堆温，超温仍会爆炸

### 流体接口配置

- **至少需要 2 个流体接口**：
  - 1 个配置为输入冷却液（使用流体管道升级）
  - 1 个配置为输出热冷却液（使用流体管道升级）
- **手动交互**：在反应堆 UI 中有 4 个流体槽，可手动注入/提取冷却液
- **升级槽**：每个流体接口有 1 个升级槽，用于配置流体管道升级

### UI 显示

热模式下的反应堆 UI 有额外显示：

- **冷却液条**（左侧，能量条外侧）：蓝色，显示输入罐的冷却液量
- **热冷却液条**（右侧，温度条外侧）：橙色，显示输出罐的热冷却液量
- **流体交互槽**：
  - 冷却液输入槽（顶部）：放入满的冷却液容器
  - 冷却液输出槽（底部）：返回空容器
  - 热冷却液输入槽（顶部）：放入空容器
  - 热冷却液输出槽（底部）：返回满的热冷却液容器

### 热模式优势与风险

| 方面 | 说明 |
|------|------|
| **优势** | 发热量翻倍，适合用于大型热能系统 |
| **优势** | 通过冷却液系统可实现更高效的散热 |
| **优势** | 可与液体热交换器等设备配合使用 |
| **风险** | 不产生 EU，需要额外的热能转电能设备 |
| **风险** | 发热量×2，堆温上升更快，爆炸风险更高 |
| **风险** | 冷却液耗尽后，散热片无法修复耐久 |
| **风险** | 堆温机制仍然存在，≥ 10,000 仍会爆炸 |

### 热模式设计原则

1. **摆法相通**：所有组件的计算逻辑在两种模式下完全相同，确保摆法可以通用
2. **统一翻倍**：发热量翻倍在 `processChambers()` 后统一处理，不修改各组件逻辑
3. **热量守恒**：散失的热量必须通过冷却液转换，不会凭空消失
4. **安全机制**：堆温机制仍然有效，超温仍会爆炸，确保设计合理的散热系统

## 元件能力概览

| 能力 | 说明 |
|------|------|
| 发电 | 燃料棒每脉冲 +1 output，周期结束转为 EU |
| 发热 | 燃料棒产生热量，优先传给邻接可储热组件，剩余进堆温 |
| 散热 | 组件蒸发自身热量，堆温降低 |
| 吸热 | 组件从堆吸收热量 |
| 热交换 | 组件与邻接/堆按温度差交换热量 |

## 容量变化时的行为

### 容量减少（如反应仓被拆）

1. **立即掉落**：`NuclearReactorBlock.neighborUpdate` 调用 `dropOverflowItems`
2. **每 tick 兜底**：`NuclearReactorBlockEntity.tick()` 也会调用 `dropOverflowItems`

### 容量增加（如放置反应仓）

仅增加可用槽位，原有物品保留。
